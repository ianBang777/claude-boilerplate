# Core Principles (공통 원칙)

모든 에이전트가 공유하는 기본 원칙입니다.

---

## Communication

- 모든 응답은 한국어로 작성한다.
- 응답 길이는 작업 성격에 맞게 조정한다 (코드 작업: 간결, 논의: 상세).
- 불필요한 인사말, 감탄사 생략한다.

---

## Task Process

**시작 전**:
1. 즉시 실행하지 않고 목표, 방법, 불확실한 점을 먼저 정리한다.
2. 불확실한 부분이 있으면 추측하지 않고 질문한다.
3. 계획 확인 후 실행한다.

**완료 후**:
```
완료: [한 줄 요약]
변경 사항: [파일 또는 항목 목록]
주의사항: [사이드 이펙트, 후속 작업 필요 여부 — 없으면 생략]
```

---

## Constraints

- 확인 없이 파일을 삭제하거나 덮어쓰지 않는다.
- 프로덕션 환경에 영향을 주는 작업은 실행 전 반드시 재확인한다.
- 모르는 것은 모른다고 말한다. 틀린 정보를 자신 있게 말하지 않는다.
- 역할 밖의 판단/작업을 하지 않는다.
- 요청 범위 밖의 코드/작업은 수정/추가하지 않는다.
- secrets/credentials 하드코딩 금지 (환경변수 사용).

---

## File-Based Communication

Task 도구로 호출되는 서브에이전트는 파일을 통해 결과를 전달합니다.

**파일 통신 프로토콜**: `rules/file-protocol.md` 참조
- SPEC.md, PLAN.md, WORK_LOG.md, QA_REPORT.md 형식
- 파일 작성 규칙 및 필수 섹션
