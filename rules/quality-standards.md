# 품질 기준

모든 작업 결과물이 충족해야 할 품질 기준입니다.

---

## 엄격한 평가 원칙

**100% 통과만 승인**:
- 모든 체크리스트 항목 **완전 충족**만 승인
- 하나라도 미달 시 → **무조건 재작업**
- 부분 승인 없음: 전체 승인 또는 전체 재작업

**금지된 표현**:
- ❌ "대체로 괜찮음"
- ❌ "거의 완성"
- ❌ "사소한 문제"
- ❌ "나중에 수정"
- ❌ "이 정도면 충분"
- ✅ "모든 항목 완전 충족" 또는 "재작업 필요"

## 코드 품질 기준

### 기본 요구사항
- [ ] 요구사항 **모두** 구현
- [ ] 누락된 항목 **전혀** 없음
- [ ] 요구사항 오해 **전혀** 없음

### 코드 품질
- [ ] 주석: **모든 함수/클래스**에 작성
- [ ] Secrets: 하드코딩 **전혀** 없음
- [ ] 컨벤션: 기존 스타일과 **완전히** 일관성
- [ ] 네이밍: 변수/함수명 명확
- [ ] 에러 처리: 예외 상황 처리

### 객관적 검증
- [ ] 테스트 실행: `npm test` / `pytest` / `cargo test` **모두 통과**
- [ ] 빌드 성공: `npm run build` / `cargo build` **에러 없음**
- [ ] 린트 통과: ESLint, Prettier **경고 없음**
- [ ] 실행 확인: 로컬 실행 후 핵심 기능 **정상 작동**

### 사이드 이펙트
- [ ] 다른 기능/파일 영향 **전혀** 없음
- [ ] 프로덕션 위험 요소 **전혀** 없음
- [ ] 후속 작업 **명확히 문서화**

## 인프라 품질 기준

### 기본 요구사항
- [ ] 요구사항 **모두** 구현
- [ ] 리소스 누락 **전혀** 없음
- [ ] 설정 오해 **전혀** 없음

### 인프라 코드 품질
- [ ] 주석: 모든 리소스 설명 작성
- [ ] Secrets: `.env` 또는 Secrets Manager 사용
- [ ] 태그: 모든 리소스에 적절한 태그
- [ ] 네이밍: 리소스명 일관성

### 객관적 검증
- [ ] Dry-run 성공: `pulumi preview` / `terraform plan` **에러 없음**
- [ ] 리소스 변경: 의도하지 않은 삭제/수정 **전혀** 없음
- [ ] 비용 영향: 예상 비용 증가분 **명시됨**
- [ ] 보안 설정: IAM, 네트워크 규칙 **적절히 제한됨**

### 보안 검토
- [ ] IAM: 최소 권한 원칙
- [ ] 네트워크: 불필요한 포트 닫힘
- [ ] 암호화: 데이터 암호화 설정
- [ ] 백업: 중요 리소스 백업 설정

### 사이드 이펙트
- [ ] 기존 리소스 영향 **전혀** 없음
- [ ] 프로덕션 다운타임 위험 **전혀** 없음
- [ ] 롤백 방법 **명확히 문서화**
