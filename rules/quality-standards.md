# 품질 기준

모든 작업 결과물이 충족해야 할 품질 기준입니다.
주로 Reviewer가 사용하지만, Worker도 사전 참조하여 품질을 높일 수 있습니다.

---

## 엄격한 평가 원칙

**100% 통과만 승인**:
- 모든 체크리스트 항목 **완전 충족**만 승인
- 하나라도 미달 시 → **무조건 재작업**
- 부분 승인 없음: 전체 승인 또는 전체 재작업

**금지된 표현**:
- ❌ "대체로 괜찮음"
- ❌ "거의 완성"
- ❌ "사소한 문제"
- ❌ "나중에 수정"
- ❌ "이 정도면 충분"
- ✅ "모든 항목 완전 충족" 또는 "재작업 필요"

## 코드 품질 기준

### 기본 요구사항
- [ ] 요구사항 **모두** 구현
- [ ] 누락된 항목 **전혀** 없음
- [ ] 요구사항 오해 **전혀** 없음

### 코드 품질
- [ ] 주석: **모든 함수/클래스**에 작성
- [ ] Secrets: 하드코딩 **전혀** 없음
- [ ] 컨벤션: 기존 스타일과 **완전히** 일관성
- [ ] 네이밍: 변수/함수명 명확
- [ ] 에러 처리: 예외 상황 처리

### 객관적 검증
- [ ] 테스트 실행: `npm test` / `pytest` / `cargo test` **모두 통과**
- [ ] 빌드 성공: `npm run build` / `cargo build` **에러 없음**
- [ ] 린트 통과: ESLint, Prettier **경고 없음**
- [ ] 실행 확인: 로컬 실행 후 핵심 기능 **정상 작동**

**예시 검증 로그**:
```bash
$ npm test
✓ All tests passed (23 tests)

$ npm run build
✓ Build completed without errors

$ npm run lint
✓ No linting errors found

$ npm run dev
✓ Server running on http://localhost:3000
✓ 기본 기능 테스트 통과 (로그인, 데이터 조회)
```

### 사이드 이펙트
- [ ] 다른 기능/파일 영향 **전혀** 없음
- [ ] 프로덕션 위험 요소 **전혀** 없음
- [ ] 후속 작업 **명확히 문서화**

## 인프라 품질 기준

### 기본 요구사항
- [ ] 요구사항 **모두** 구현
- [ ] 리소스 누락 **전혀** 없음
- [ ] 설정 오해 **전혀** 없음

### 인프라 코드 품질
- [ ] 주석: 모든 리소스 설명 작성
- [ ] Secrets: `.env` 또는 Secrets Manager 사용
- [ ] 태그: 모든 리소스에 적절한 태그
- [ ] 네이밍: 리소스명 일관성

### 객관적 검증
- [ ] Dry-run 성공: `pulumi preview` / `terraform plan` **에러 없음**
- [ ] 리소스 변경: 의도하지 않은 삭제/수정 **전혀** 없음
- [ ] 비용 영향: 예상 비용 증가분 **명시됨**
- [ ] 보안 설정: IAM, 네트워크 규칙 **적절히 제한됨**

 +   ├─ aws:ec2:Vpc            main        create
 +   ├─ aws:ec2:Subnet         public      create
 +   └─ aws:ec2:SecurityGroup  web         create

Resources:
    + 4 to create

Estimated cost: +$50/month

### 보안 검토
- [ ] IAM: 최소 권한 원칙
- [ ] 네트워크: 불필요한 포트 닫힘
- [ ] 암호화: 데이터 암호화 설정
- [ ] 백업: 중요 리소스 백업 설정

### 사이드 이펙트
- [ ] 기존 리소스 영향 **전혀** 없음
- [ ] 프로덕션 다운타임 위험 **전혀** 없음
- [ ] 롤백 방법 **명확히 문서화**

## 문서 품질 기준

### 기본 요구사항
- [ ] 요구사항 **모두** 포함
- [ ] 누락된 섹션 **전혀** 없음
- [ ] 내용 오해 **전혀** 없음

### 문서 품질
- [ ] 구조: 명확한 계층 (H1, H2, H3)
- [ ] 가독성: 문장 간결, 불필요한 수식 제거
- [ ] 일관성: 용어 통일, 스타일 일관
- [ ] 완성도: 미완성 섹션 없음

### 객관적 검증
- [ ] 링크 검증: 모든 내부/외부 링크 **정상 작동**
- [ ] 코드 샘플: 모든 코드 예시 **실제 실행 가능**
- [ ] 스펠링: 맞춤법 검사 통과
- [ ] 포맷팅: Markdown 형식 올바름

**예시 검증**:
```bash
# 링크 검증
✓ README.md: 모든 링크 정상 (0 broken links)

# 코드 샘플 실행
$ node examples/basic.js
✓ 정상 실행

# 맞춤법 검사
✓ 0 spelling errors
```

### 대상 독자 적합성
- [ ] 기술 수준 맞춤
- [ ] 필요한 배경 설명 포함
- [ ] 불필요한 전문 용어 최소화

## 검증 결과 기록 형식

### QA_REPORT.md 형식

**승인 시**:
```markdown
# 품질 검토 보고서

## 검토 결과
✅ 승인

## 체크리스트 결과
- 요구사항 충족: ✅ 3/3 통과
- 품질 기준: ✅ 5/5 통과
- 사이드 이펙트: ✅ 3/3 통과
- 객관적 검증: ✅ 모두 통과

## 객관적 검증 상세
### 테스트 실행
\`\`\`bash
$ npm test
✓ All tests passed (23 tests)
\`\`\`

### 빌드 결과
\`\`\`bash
$ npm run build
✓ Build completed without errors
\`\`\`

### 실행 확인
- 로그인 기능: ✅ 정상
- 데이터 조회: ✅ 정상
- API 응답: ✅ 정상 (평균 200ms)
```

**재작업 필요 시**:
```markdown
# 품질 검토 보고서

## 검토 결과
❌ 재작업 필요

## 체크리스트 결과
- 요구사항 충족: ❌ 2/3 통과
- 품질 기준: ✅ 5/5 통과
- 사이드 이펙트: ✅ 3/3 통과
- 객관적 검증: ❌ 테스트 실패

## 미달 항목
1. **요구사항 누락**: "사용자 인증" 기능 미구현
2. **테스트 실패**: `test/auth.test.ts:42` 실패

## 객관적 검증 상세
### 테스트 실행
\`\`\`bash
$ npm test
✗ 1 test failed
  ✓ user.test.ts (10 tests)
  ✗ auth.test.ts:42 - Expected 200, got 401
\`\`\`

## 재작업 지시
1. Worker → 사용자 인증 기능 추가 (SPEC.md 3.2절 참고)
2. Worker → `test/auth.test.ts:42` 실패 원인 수정
3. 수정 후 `npm test` 모두 통과 확인
```

## Reviewer 준수사항

### 반드시 할 것
1. **객관적 검증 실행**
   - 테스트/빌드 명령어 실제 실행
   - 결과 로그를 QA_REPORT.md에 포함

2. **체크리스트 100% 확인**
   - 모든 항목 개별 체크
   - 하나라도 미달 시 재작업

3. **구체적 피드백**
   - "주석이 부족함" ❌
   - "src/auth.ts:23 함수에 주석 없음" ✅

### 절대 하지 말 것
1. **모호한 승인**
   - "대체로 괜찮음"
   - "사소한 문제"

2. **부분 승인**
   - "90% 완성"
   - "나중에 고치면 됨"

3. **검증 생략**
   - 테스트 실행 없이 코드만 읽기
   - 빌드 확인 없이 승인

## Worker 사전 참조

Worker는 작업 시작 전 이 문서를 참조하여:
- 품질 기준 사전 파악
- 검증 방법 미리 준비
- 재작업 최소화

**권장 프로세스**:
1. SPEC.md 읽기
2. rules/quality-standards.md 읽기
3. 품질 기준 충족하도록 작업
4. 자체 검증 (테스트 실행 등)
5. WORK_LOG.md 작성 (검증 결과 포함)
