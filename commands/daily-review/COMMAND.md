# /daily-review

오늘 작업한 변경사항을 요약하고, 배포 전 확인이 필요한 포인트를 정리합니다.

## 실행 순서

### 1단계: 오늘 변경사항 수집

```bash
# 오늘 커밋된 내역
git log --since="today 00:00" --oneline

# 아직 커밋되지 않은 변경사항
git diff HEAD
git status
```

### 2단계: 변경사항 분석 (병렬 실행)

변경된 파일 각각에 대해 다음을 파악합니다:

- 어떤 기능이 추가/수정/삭제되었는가
- 다른 컴포넌트·모듈에 영향을 주는 변경인가
- 테스트가 필요한 부분이 있는가

### 3단계: 리뷰 포인트 도출

`/code-review` 체크리스트 기준으로 오늘 변경사항을 검토합니다.

### 4단계: 리포트 출력

아래 형식으로 출력합니다:

```markdown
## 데일리 리뷰 — [날짜]

### 오늘 작업 요약
- [커밋/변경 단위로 한 줄 요약]

### 확인 필요
- [BLOCK] ...
- [WARN] ...

### 내일 이어서 할 것
- [미완료 작업, 남은 TODO 등]

### 전체 평가
[오늘 작업의 완성도, 리스크, 한 줄 총평]
```

## 옵션

인자를 넘기면 범위를 조정할 수 있습니다:

```
/daily-review --branch feature/my-feature   # 특정 브랜치 기준
/daily-review --since yesterday             # 어제부터 오늘까지
/daily-review --file src/components/Foo.tsx # 특정 파일만
```
