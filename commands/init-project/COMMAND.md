# /init-project

새로운 프로젝트에 처음 진입했을 때, 코드베이스를 분석하고 프로젝트 루트에 `CLAUDE.md`를 자동 생성합니다.

## 실행 순서

### 1단계: 코드베이스 파악 (병렬 실행)

아래 항목을 동시에 조사합니다:

- 프로젝트 구조 (`Glob("**/*", depth: 3)`)
- 패키지 매니저 및 의존성 (`package.json` / `build.gradle.kts` / `pom.xml`)
- 기존 설정 파일 (`.eslintrc`, `tsconfig.json`, `application.yml` 등)
- 기존 `CLAUDE.md` 또는 `README.md` 존재 여부
- 엔트리포인트 파일 (주요 라우트, 컨트롤러, 컴포넌트)

### 2단계: 분석 결과 정리

다음 항목을 파악합니다:

- **기술 스택**: 프레임워크, 주요 라이브러리, 버전
- **폴더 구조 패턴**: 기능별 / 레이어별 / 도메인별
- **코드 컨벤션**: 네이밍 규칙, import 방식, 상태 관리 방식
- **주의해야 할 특이사항**: 레거시 코드, 특수한 설정, 알려진 이슈

### 3단계: CLAUDE.md 생성

아래 형식으로 프로젝트 루트에 `CLAUDE.md`를 작성합니다:

```markdown
# [프로젝트명]

## 기술 스택
...

## 폴더 구조
...

## 개발 규칙
...

## 주요 명령어
...

## 주의사항
...
```

### 4단계: 확인

생성한 `CLAUDE.md`를 사용자에게 보여주고 수정이 필요한 부분을 확인합니다.

## 원칙

- 추측하지 않습니다. 코드에서 확인된 내용만 작성합니다.
- 간결하게 작성합니다. `CLAUDE.md`는 레퍼런스 문서가 아닌 작업 컨텍스트 파일입니다.
- 기존 `CLAUDE.md`가 있으면 덮어쓰지 않고 병합 여부를 먼저 확인합니다.
