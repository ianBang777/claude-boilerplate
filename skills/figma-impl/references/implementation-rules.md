# 구현 우선순위 및 변환 규칙

다음 우선순위로 구현합니다.

1. **기존 코드 재사용** — 새로 만들기 전에 프로젝트에 이미 있는 컴포넌트·토큰을 먼저 활용
2. **프로젝트 컨벤션 준수** — Figma 레퍼런스 코드를 그대로 붙여 넣지 않고 프로젝트 스택에 맞게 변환; 디자인에 아이콘이 필요한 경우 프로젝트에서 사용 중인 아이콘 라이브러리를 확인하여 추가
3. **타입 안전성** — TypeScript props 타입 명시, nullable 여부 반영

## 변환 규칙

- `get_variable_defs`로 수집한 Figma 토큰 → 프로젝트 CSS 변수 또는 토큰과 대조하여 매핑
- Figma 토큰에 대응하는 프로젝트 토큰이 없으면, 하드코딩 대신 누락 토큰을 명시하고 확인을 요청
- Figma의 절대 수치(px)는 토큰으로 대체 가능한지 먼저 확인하고, 불가능한 경우에만 수치 사용
- Code Connect 스니펫이 반환된 경우 → 해당 컴포넌트를 직접 사용
- 시각적 추측(스크린샷 참조, 색상 눈대중)으로 값을 결정하지 않음
