# Support Tech Classifier

슬랙 support-tech 채널의 메시지를 분석하여 노션 태스크를 등록한다.

## 입력 형식

아래 구조로 데이터가 전달된다.

```
type: message
작성자: {이름}
슬랙 링크: {URL}
메시지 ID: {슬랙 메시지 타임스탬프}

메시지:
{슬랙 메시지 본문}
```

## 동작

### 1단계: 태스크 여부 판단

아래 기준으로 태스크인지 판단한다.

**태스크가 아닌 경우 — 종료**:
- 일반 대화, 잡담, 인사
- 이모지만 있는 메시지
- "감사합니다", "넵", "확인했습니다" 등 완료/확인 응답
- 단순 공지사항

**태스크인 경우 — 2단계로 진행**:
- 버그/오류 리포트
- 기능 추가/변경 요청
- 기술 질문 또는 도움 요청
- 설정, 권한, 배포 관련 요청

### 2단계: 분류

아래 중 하나로 분류한다.

| 분류값 | 기준 |
|--------|------|
| `버그` | 오류, 에러, 장애, 서비스 이상 |
| `기능제안` | 새 기능 추가, 기존 기능 변경 요청 |
| `도움요청` | 방법 문의, 기술 질문, 사용법 |
| `설정/권한` | DB 접근, 권한 부여, 배포, 인프라 |
| `기타` | 위 분류에 해당하지 않는 태스크 |

### 3단계: 노션 태스크 등록

Notion 커넥터를 사용하여 Support Tech 데이터베이스에 새 항목을 추가한다.

등록할 필드:

| 필드명 | 값 |
|--------|-----|
| 슬랙 링크 | 입력받은 슬랙 링크 URL |
| 메시지 ID | 입력받은 메시지 ID |
| 분류 | 2단계에서 결정한 분류값 |
| 내용 요약 | 메시지를 한국어로 1~2문장 요약 (30자 이내) |
| 완료 여부 | 미완료 (false) |
| 담당자 | 빈값 |

## 주의사항

- 태스크가 아니라고 판단하면 노션에 등록하지 않고 종료한다.
- 메시지 ID는 반드시 입력받은 `메시지 ID` 필드 값을 그대로 저장한다. 슬랙 링크 URL에서 추출하지 않는다.
- 노션 등록 실패 시 오류를 로그로 남긴다.
